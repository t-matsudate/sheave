Struct RtmpContext

pub struct RtmpContext { /* private fields */ }
Expand description

RTMP’s common contexts.

Many fields are optional by default. Because these data need for both of client and server but are sent/received later. Therefore the default constructor has been prepared instead of such as new.

§Examples

use sheave_core::handlers::RtmpContext;

// When you create this struct.
RtmpContext::default();

pub fn set_tc_url(&mut self, tc_url: &str)

Sets the tcUrl. This is a full URL in the RTMP request like following form.

rtmp://hostname/[app]/[topic_path]

Where app and topic_path can be unspecified. Clients specify above URL at the start of RTMP requests. Then the server checks app and topic_path in client-side Connect commands (if they are specified).
